  1. DecisionEngine.make_decision("CVE-2021-1234", 0.85)
  2. RecommendationEngine.generate_recommendations(...)
  3. Store DecisionOutput in database
  4. Display in UI for approval

WORKFLOW 2: Batch Vulnerability Processing
  1. DecisionEngine.batch_decide(cve_list, risk_scores)
  2. PriorityCalculatorEngine.batch_prioritize(batch_decisions)
  3. Store all results in database
  4. Display priority-ranked list in UI

WORKFLOW 3: Compliance Reporting
  1. ComplianceMapperEngine.detect_applicable_frameworks(industry, role)
  2. ComplianceMapperEngine.assess_compliance_impact(cve_id, frameworks)
  3. ComplianceMapperEngine.generate_compliance_report(assessments)
  4. Export report for compliance team

WORKFLOW 4: Strategy Selection
  1. RemediationStrategyEngine.determine_strategy(full parameters)
  2. Review strategy recommendations
  3. Approve or override strategy
  4. Execute selected approach

WORKFLOW 5: Rollback Risk Evaluation
  1. RollbackRiskAssessor.assess_overall_rollback_risk(kb_id, system_info)
  2. Review risk level and feasibility
  3. Adjust strategy if rollback too risky
  4. Proceed with safer approach

# ============================================================================
# TROUBLESHOOTING GUIDE
# ============================================================================

Database won't initialize:
  → Check dev_db.sqlite and runtime_db.sqlite exist
  → Verify paths are correct in config
  → Check SQLite write permissions

Module import errors:
  → Verify modules are in src/riskengine/
  → Check Python path includes src/
  → Review Python version (3.8+)

Decision engine returns None:
  → Check CVE exists in dev_db
  → Verify system data exists in runtime_db
  → Review configuration JSON loaded correctly
  → Check logs for SQL errors

PowerShell scripts not executable:
  → Review generated script syntax
  → Check Windows PowerShell execution policy
  → Verify path expansion works correctly
  → Review script validation steps

Compliance not detected:
  → Check system_role, industry, data_classification set
  → Verify compliance_mapper config loaded
  → Review applicable_frameworks list
  → Check framework definitions in code

Performance issues:
  → Verify database indexes created
  → Check query performance with EXPLAIN
  → Consider batch processing instead of one-by-one
  → Review logging level (reduce DEBUG verbosity)
