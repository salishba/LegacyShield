## Supported Compliance Frameworks:

1. HIPAA (Health Insurance Portability and Accountability Act)
   - Applies to: Healthcare organizations, healthcare providers, health plans
   - Key regulations: 164.308(a)(3)(ii)(B), 164.308(a)(5)(ii)(A)
   - CVE Urgency: CRITICAL→4hrs, HIGH→24hrs, MEDIUM→72hrs
   
2. PCI DSS (Payment Card Industry Data Security Standard)
   - Applies to: Merchants, processors, issuers handling payment cards
   - Sections: 6.2 (Patch management), 11.2 (Vulnerability scanning)
   - CVE Urgency: CRITICAL→30 days, HIGH→30 days
   
3. NIST (National Institute of Standards and Technology)
   - Applies to: Federal government, critical infrastructure, defense
   - Controls: SI-2 (Flaw remediation), RA-2 (Security assessments)
   - CVE Urgency: CRITICAL→7 days, HIGH→30 days
   
4. SOC 2 (Service Organization Control)
   - Applies to: Cloud providers, SaaS platforms, service providers
   - Criteria: CC6.1 (Access controls), CC7.2 (Change management)
   - CVE Urgency: CRITICAL→7 days, HIGH→30 days
   
5. ISO 27001 (Information Security Management)
   - Applies to: Any organization managing sensitive information
   - Control: A.12.6.1 (Technical vulnerability management)
   - CVE Urgency: CRITICAL→14 days, HIGH→30 days
   
6. GDPR (General Data Protection Regulation)
   - Applies to: EU organizations, orgs handling EU resident data
   - Article: 32 (Security of processing)
   - CVE Urgency: CRITICAL→4 hours (data breach risk), HIGH→24 hours
   
7. FedRAMP (Federal Risk and Authorization Management Program)
   - Applies to: Cloud providers serving US federal agencies
   - Enhancement: IA-5 (Authentication controls)
   - CVE Urgency: CRITICAL→15 days, HIGH→30 days

## Component Scorers (all 0.0-1.0):

1. Severity Score (25% weight in final priority)
   - CVSS base score normalization
   - Boost for weaponized/exploited vulnerabilities
   - Boost for ransomware association
   - PoC availability multiplier
   - Formula: (cvss/10) × exploit_boost × ransomware_boost × poc_boost
   
2. Exploitability Score (20% weight)
   - EPSS probability score
   - Attack complexity modifier (easy=1.0, hard=0.3)
   - Privileges required (none=1.0, required=0.5)
   - User interaction required (no=1.0, yes=0.3)
   - Network accessible (yes=1.0, local=0.5)
   - Formula: epss × complexity × privileges × interaction × network
   
3. Business Impact Score (25% weight)
   - System role criticality: DC=1.0, DB=0.8, Web=0.7, WS=0.4
   - Data classification multiplier: restricted=1.0, confidential=0.8, internal=0.5
   - Compliance violations multiplier: sum per framework
   - Formula: role_score × data_score × (1 + compliance_count×0.2)
   
4. System Density Score (15% weight)
   - Spread ratio: affected_systems / total_systems
   - Critical infrastructure multiplier: 1.5 if critical
   - DMZ deployment multiplier: 2.0 if exposed
   - Network exposure: external=1.0, dmz=0.8, internal=0.4
   - Formula: spread × infrastructure × dmz × network
   
5. SLA Urgency Score (15% weight)
   - Time to SLA deadline calculation
   - Breached: 1.0
   - At risk (< 25% time remaining): 0.9
   - Soon (25-50% time remaining): 0.7
   - Normal (> 50% time remaining): 0.4
   - Returns tuple: (score, status_string)

LOCATION: src/riskengine/rollback_risk_assessor.py
PURPOSE: Comprehensive rollback risk assessment and feasibility analysis
LINES: 720+
PRODUCTION READY: ✓ YES - 6-component weighted model

## Risk Assessment Components:

1. Patch Maturity Risk (30% weight)
   - Score by age: 0-7 days=0.1, 8-14=0.2, 15-30=0.5, 31-60=0.7, 61+=0.9
   - Newer patches are more risky (less field-tested)
   
2. Adoption Rate Risk (15% weight)
   - Environment adoption %: 80%+=0.1, 60-80%=0.3, 40-60%=0.5, 20-40%=0.7, <20%=0.9
   - Higher adoption = lower risk (more testing has occurred)
   
3. Known Issues Risk (30% weight)
   - Query known_kb_issues database
   - Critical issues ×0.3, High ×0.15, Medium ×0.05, Low ×0.02
   - Sum all: issue_risk
   
4. System Criticality Risk (15% weight)
   - Role multiplier: DC=0.9, DB=0.8, Web=0.7, File=0.6, WS=0.3
   - Uptime factor: 700+ days=1.0, 500-700=0.9, 100-500=0.7, <100=0.5
   - Required services: high_count = higher risk
   - Backup age: <24hrs=0.1, 1-7 days=0.4, 8-30days=0.7, >30=0.9
   - Can disconnect: True=0.3, False=0.8
   
5. Application Compatibility Risk (7% weight)
   - Query known_incompatibilities table
   - Critical incompats ×0.5, High ×0.25, Medium ×0.10
   
6. Dependency Complexity Risk (3% weight)
   - Affected system components count
   - Service dependencies count
