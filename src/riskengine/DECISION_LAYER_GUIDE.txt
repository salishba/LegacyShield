# ============================================================================
# DECISION LAYER COMPONENTS
# ============================================================================

## Components Created:

1. **decision_engine.py** (Core Orchestration)
   - Main entry point: DecisionEngine class
   - Fetches vulnerability and system data at runtime
   - Orchestrates all component engines
   - Makes complete decisions for vulnerabilities
   - Batch processing support
   - NO HARDCODED LOGIC - all configuration loaded dynamically

2. **recommendation_engine.py** (Actionable Recommendations)
   - Generates specific, executable recommendations
   - PowerShell script generation for patch installation
   - Registry hardening procedures
   - Service disable procedures
   - File replacement guides
   - Rollback procedures with automation
   - Test validation steps

3. **remediation_strategy.py** (Strategic Planning)
   - Determines optimal remediation approach
   - Evaluates all strategy options with scoring
   - Patch maturity assessment
   - Patch adoption tracking
   - Known issues database lookup
   - Business impact analysis
   - Generates complete execution strategies

4. **compliance_mapper.py** (Regulatory Compliance)
   - Maps to 6 compliance frameworks:
     * HIPAA (Healthcare)
     * PCI DSS (Payment Card Industry)
     * NIST (Federal/Critical Infrastructure)
     * SOC 2 (Service Providers)
     * ISO 27001 (Information Security)
     * GDPR (Data Privacy)
   - SLA calculation per framework
   - Violation detection
   - Audit trail requirements
   - Compliance risk scoring

5. **priority_calculator.py** (Batch Prioritization)
   - Dynamic vulnerability prioritization
   - 5-component scoring model:
     * Severity (CVSS, exploitability, ransomware)
     * Exploitability (EPSS, attack complexity, access)
     * Business Impact (system role, data classification)
     * System Density (affected systems, criticality)
     * SLA Urgency (time to deadline)
   - Batch processing with ranking
   - Resource-aware scheduling
   - Historical trending support

6. **rollback_risk_assessor.py** (Patch Risk Analysis)
   - Assesses rollback failure probability
   - Components:
     * Patch maturity and adoption
     * Known issues lookup
     * System criticality
     * Backup availability
     * Application compatibility
   - Rollback feasibility validation
   - Risk mitigation recommendations
   - Estimated rollback time calculation

7. **database_schema.py** (Persistence Layer)
   - Complete SQL schemas for all Decision Layer tables
   - Audit trail tables for compliance
   - Historical tracking tables
   - Performance indexes
   - Schema validation and initialization
   - Export/backup utilities

# ============================================================================
# ARCHITECTURE OVERVIEW
# ============================================================================

## Data Flow:

Detection Layer (Agent)
        ↓
        └→ System Context: OS version, KBs, services, ports
        └→ File Hashes: Component versions for detection
        
Vulnerability Data (CVE Database)
        ↓
        └→ CVSS, EPSS, exploitation status, ransomware association
        └→ Affected components, patches, mitigations
        
Scoring Engine (Existing)
        ↓
        └→ r_score, a_score, c_score
        └→ Risk probability and exploitability
        
DECISION LAYER (NEW)
    ↓
    ├→ decision_engine.py
    │  ├→ Fetch all runtime data
    │  ├→ Adjust risk for system context
    │  ├→ Determine action and urgency
    │  └→ Calculate rollback risk
    │
    ├→ compliance_mapper.py
    │  ├→ Detect applicable frameworks
    │  ├→ Calculate SLA requirements
    │  └→ Generate compliance impacts
    │
    ├→ priority_calculator.py
    │  ├→ Score vulnerability components
    │  ├→ Calculate overall priority
    │  └→ Batch rank all vulnerabilities
    │
    ├→ remediation_strategy.py
    │  ├→ Evaluate all strategy options
    │  ├→ Assess patch maturity/adoption
    │  └→ Generate execution plan
    │
    ├→ recommendation_engine.py
    │  ├→ Generate specific recommendations
    │  ├→ Create PowerShell scripts
    │  └→ Provide step-by-step procedures
    │
    └→ rollback_risk_assessor.py
       ├→ Assess all risk factors
       ├→ Validate rollback capability
       └→ Provide mitigation steps
            ↓
        Decision Output
            ↓
        Recommendation Output
            ↓
        Strategy Output
            ↓
        [Execution Layer - (Phase 2)]
            ↓
        [Rollback Layer - (Phase 3)]

## 2. Make Decision for Single Vulnerability

```python
from src.riskengine.decision_engine import DecisionEngine

# Create engine
engine = DecisionEngine(
    runtime_db_path="runtime_scan.sqlite",
    dev_db_path="dev_db.sqlite"
)

# Make decision for CVE
decision = engine.make_decision(
    cve_id="CVE-2021-1234",
    r_score=0.85  # From scoring engine
)

    print(decision.human_summary)
    print(f"Recommended Action: {decision.recommended_action.value}")
    print(f"Risk Score: {decision.risk_score:.1%}")
```

## 3. Batch Process Multiple Vulnerabilities

```python
# Batch process
cve_ids = ["CVE-2021-1234", "CVE-2021-5678", "CVE-2022-9000"]
r_scores = {
    "CVE-2021-1234": 0.85,
    "CVE-2021-5678": 0.60,
    "CVE-2022-9000": 0.92
}

batch_result = engine.batch_decide(cve_ids, r_scores)

print(f"Processed: {batch_result.processed_vulnerabilities}")
print(f"CRITICAL: {batch_result.critical_count}")
print(f"HIGH: {batch_result.high_count}")

# Decisions sorted by priority
for decision in batch_result.decisions:
    print(f"{decision.priority_level}: {decision.vulnerability.cve_id}")
```
