# COMPLETE WORKFLOW & INTEGRATION

## Full Remediation Workflow:

1. DECISION LAYER OUTPUT
   Input: CVE, vulnerability analysis from decision_engine.py
   Output: DecisionOutput with priority, risk score, compliance impact
   
2. COMPONENT MAPPING
   Input: DecisionOutput + CVE details
   Process:
     a. Map CVE to affected components (kernel32.dll, srv2.sys, rpcss.exe, etc.)
     b. Query system inventory for affected systems
     c. Map components to applicable hardening controls
   Output: VulnerabilityComponentMap with component list and control IDs

3. CATALOGUE CONTROL SELECTION
   Input: VulnerabilityComponentMap + Control IDs
   Process:
     a. Load mitigation catalogues (5 control types)
     b. Find controls by ID from catalogue
     c. Verify OS applicability
     d. Extract enforcement methods and validation checks
   Output: List of RemediationControl objects with full procedures

4. EXECUTION PLAN GENERATION
   Input: Components + Controls + Affected Systems
   Process:
     a. Build execution phases (discovery, validation, deployment, verification)
     b. Generate commands for each control on each system
     c. Create reasoning/explainability model
     d. Assign effort estimates
   Output: ExecutionPlan with all steps and reasoning

5. VALIDATION & TESTING
   Input: ExecutionPlan
   Process:
     a. Check system readiness (admin, backup, network, etc.)
     b. Verify backup availability and freshness
     c. Run sandbox test on test system (optional)
     d. Validate rollback capability
   Output: ValidationReport with safe-to-deploy recommendation

6. EXECUTION (Dry-Run or Real)
   Input: ExecutionPlan + ExecutionMode (DRY_RUN, SANDBOX, PRODUCTION)
   Process:
     a. Execute phase sequence (discovery → validation → deployment → verification)
     b. Track execution steps and status
     c. Log all outputs and errors
     d. Handle failures and rollback
   Output: Execution results with status per step

7. AUDIT & LOGGING
   Input: All execution results
   Output: Execution logs stored in execution_log.sqlite
   Audit trail: User, timestamp, action, system, result

```python
from src.riskengine.decision_engine import DecisionEngine
from src.services.mitigation_catalogue_loader import MitigationCatalogueLoader
from src.services.component_mapper import ComponentMapper
from src.services.execution_engine import ExecutionEngine, ExecutionMode
from src.services.test_validator import TestValidator

# STEP 1: Get decision from Decision Layer
decision_engine = DecisionEngine("runtime_scan.sqlite", "dev_db.sqlite")
decision = decision_engine.make_decision("CVE-2021-1234", r_score=0.85)

print(f"Decision: {decision.priority_level} priority")
print(f"Risk Score: {decision.risk_score:.1%}")
print(f"Compliance Violations: {decision.compliance_violations}")

# STEP 2: Create Execution Engine and load catalogues
exec_engine = ExecutionEngine(
    catalogue_dir="src/catalogues",
    dev_db_path="dev_db.sqlite",
    runtime_db_path="runtime_scan.sqlite"
)
exec_engine.load_catalogues()

# STEP 3: Generate remediation plan
plan = exec_engine.generate_remediation_plan(
    cve_id="CVE-2021-1234",
    decision_output=decision.__dict__,
    vulnerability_components=["SMB", "RCE", "wormable"],
    affected_os="Windows 10",
    mode=ExecutionMode.DRY_RUN  # Start with dry-run
)

print(f"\nRemediation Plan: {plan.plan_id}")
print(f"  Controls to Apply: {len(plan.controls)}")
print(f"  Affected Systems: {len(plan.affected_systems)}")
print(f"  Execution Phases: {len(plan.phases)}")

# Show control details
for control in plan.controls:
    print(f"  - {control.catalog_id}: {control.name} ({control.mitigation_type.value})")

# STEP 4: Validate before execution
validator = TestValidator()

# Check readiness for each affected system
readiness_checks = {}
for system_id in plan.affected_systems:
    readiness = validator.check_system_readiness(
        system_id=system_id,
        os_version="Windows 10",
        system_info={
            "admin_access": True,
            "backup_available": True,
            "backup_age_hours": 12,
            "network_available": True
        }
    )
    readiness_checks[system_id] = readiness
    
    if readiness.ready_for_execution:
        print(f"✓ {system_id}: Ready for execution")
    else:
        print(f"✗ {system_id}: Not ready - address issues first")
        for check in readiness.checks:
            if not check.passed:
                print(f"    - {check.name}: {check.message}")

# Run sandbox test (optional)
sandbox_results = []
if len(plan.affected_systems) > 0:
    # Use first system as test
    test_result = validator.run_sandbox_test(
        plan_id=plan.plan_id,
        cve_id="CVE-2021-1234",
        test_system_id=f"{plan.affected_systems[0]}-TEST",
        controls_to_test=[
            {"name": c.name, "type": c.mitigation_type.value}
            for c in plan.controls
        ]
    )
    sandbox_results.append(test_result)
    print(f"\n✓ Sandbox Test: {'PASSED' if test_result.all_validations_passed else 'FAILED'}")

# Generate validation report
validation_report = validator.generate_validation_report(
    plan_id=plan.plan_id,
    cve_id="CVE-2021-1234",
    readiness_checks=readiness_checks,
    sandbox_results=sandbox_results
)

print(f"\nValidation Report:")
print(f"  Safe to Deploy: {'✓ YES' if validation_report.safe_to_deploy else '✗ NO'}")
print(f"  Readiness: {validation_report.deployment_readiness_percentage:.0f}%")
print(f"  Critical Issues: {validation_report.critical_issues}")
for rec in validation_report.recommended_actions:
    print(f"  {rec}")

# STEP 5: Execute (dry-run first)
print(f"\n--- Executing (DRY-RUN MODE) ---")
dry_run_results = exec_engine.execute_plan(
    plan=plan,
    mode=ExecutionMode.DRY_RUN,
    dry_run=True
)

print(f"Execution Results:")
print(f"  Total Steps: {dry_run_results['summary']['total_steps']}")
print(f"  Completed: {dry_run_results['summary']['completed']}")
print(f"  Success Rate: {dry_run_results['summary']['success_rate']:.0%}")

# If dry-run successful and validation passed, can proceed to production
if (dry_run_results['summary']['success_rate'] == 1.0 and 
    validation_report.safe_to_deploy):
    print("\n✓ Ready for production deployment")
    # production_results = exec_engine.execute_plan(plan, mode=ExecutionMode.PRODUCTION, dry_run=False)
else:
    print("\n✗ Should not deploy - review validation issues")
```

# CONTROL TYPES & MITIGATION CATEGORIES
# ============================================================================

## Mitigation Type Categories (Enum: MitigationType):

1. **APPLICATION** (APP-01 through APP-04)
   Controls for disabling/hardening applications
   Examples:
     - APP-01: Disable Windows PowerShell 2.0
     - APP-02: Disable Internet Explorer 
     - APP-03: Application whitelisting
     - APP-04: Script Execution Policy

2. **AUTHENTICATION** (AUTH-01 through AUTH-04)
   Controls for hardening authentication mechanisms
   Examples:
     - AUTH-01: Disable LM hash storage
     - AUTH-02: Enable NTLMv2 enforcement
     - AUTH-03: Restrict NTLM authentication
     - AUTH-04: Enforce MFA/strong password policy

3. **NETWORK** (NET-01 through NET-11)
   Controls for disabling/hardening network services
   Examples:
     - NET-01: Disable SMBv1 (client & server)
     - NET-02: Enable SMB encryption
     - NET-03: Disable NetBIOS over TCP/IP
     - NET-04: Disable RPC endpoints
     - NET-05: Firewall rules
     - NET-06: Port hardening
     - ... (up to NET-11)

4. **REGISTRY** (REG-01 through REG-06)
   Registry-based hardening controls
   Examples:
     - REG-01: Disable AutoRun/AutoPlay
     - REG-02: Restrict anonymous enumeration
     - REG-03: Disable Remote Registry
     - REG-04: UAC hardening
     - REG-05: LSA protection
     - REG-06: LSASS credential guard

5. **SERVICES** (SVC-01 through SVC-06, if available)
   Service-level controls
   Examples:
     - SVC-01: Disable RPC service
     - SVC-02: Disable Print Spooler
     - SVC-03: Disable Remote Desktop
     - SVC-04: Disable WinRM
     - SVC-05: Disable scheduled tasks
     - SVC-06: Service principal name hardening

## Component Types (Enum: ComponentType):

- **DLL**: Dynamic library (.dll)
  Example: kernel32.dll, rpcrt4.dll, ntlm.dll
  
- **EXE**: Executable (.exe)
  Example: powershell.exe, lsass.exe, spoolsv.exe
  
- **DRIVER**: System driver (.sys)
  Example: srv2.sys, ndis.sys
  
- **REGISTRY_KEY**: Registry key/value
  Example: HKLM\System\CurrentControlSet\Control\Lsa
  
- **SERVICE**: System/user service
  Example: RpcSs, LanmanServer, spooler
  
- **FEATURE**: Windows optional feature
  Example: MicrosoftWindowsPowerShellV2Root, SMB1Protocol
  
- **CONFIG**: Configuration file
  Example: Group Policy files, service config
  
- **LIBRARY**: Software library/module
  Example: .NET framework, runtime libraries

# ============================================================================
# TEST MODE & DRY-RUN FEATURES
# ============================================================================

## Execution Modes:

1. **DRY-RUN Mode**
   - Simulates execution without making changes
   - Shows what would be executed
   - Useful for initial validation
   - No system impact

2. **SANDBOX Mode**
   - Runs on isolated test system
   - Creates VM clone or container
   - Full execution with rollback capability
   - Tests rollback procedures

3. **STAGED Mode**
   - Deploys to subset of systems (e.g., 10%)
   - Monitors for issues
   - Escalates to full deployment if successful
   - Risk mitigation for large rollouts

4. **PRODUCTION Mode**
   - Full deployment to all affected systems
   - Real-time monitoring
   - Automatic rollback on critical failures

## Validation Levels:

1. **CRITICAL** (must pass)
   - Admin access
   - Backup availability
   - Network connectivity
   - OS compatibility

2. **WARNING** (should pass)
   - Backup currency (< 24 hours)
   - Antivirus compatibility
   - Maintenance window availability

3. **INFO** (informational)
   - Uptime metrics
   - Service status
   - Current patch levels
